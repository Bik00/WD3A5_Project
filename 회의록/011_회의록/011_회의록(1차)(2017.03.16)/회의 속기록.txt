가구 판매업체의 흐름, UI => 각각의 흐름과 UI

서비스의 핵심적인 기능을 정의해야함.

기능을 놓고 내용을 갖다 붙이면 구현을 할 때 해야될 것이 굉장히 많음.
그러면, 고객 입장에서는 자신이 구매하고자 하는 가구를 배치할 수 있는 서비스이다.
이 서비스의 이름은? 
예를 들면, 가구 판매업자의 경우에는 복합쇼핑몰을 몰앤몰 이런식으로

그다음으로, 이해당사자들이 해야할 일들, 필요한 것들을 세분화 해야 함.
나중에 스토리보드를 제작할 때에는 작은 팝업창 하나하나까지 모두 표현해야 함.

제일 먼저, 각자의 롤이 나와야 함. 크게 프로세스를 잡고, 방향성을 잡아야 함.
그다음 세분화해서 스코프를 정하기.

분석 프로세스, 주요 서비스(카테고리화), 기능
- 카테고리화 시키는 것은 이해 당사자 측에서 or 기능 면에서
-> 이렇게 해서 양이 정해지면 역할 분담을 해야 함.
=> 한 눈에 볼 수 있는 기획서를 작성할 수 있음.

만약 내가 가구판매업체라면 가구를 어떻게 등록할 것인가?
가구 업체를 승인해야 하는 등의 시스템 관리자가 필요함.

실제로 시행하는 사람과 이야기 해보고 명확히 해야 함.

뭐가 합리적인가?

도면이 여러개라는 것은 아파트별로 평수가 다르기 때문이기도 하지만,
아파트가 여러 개 일 수도 있다. 
그리고, 시행사가 여러 개일 수도 있다.

관계들을 명확히하고 플로우를 정해놓고 이 플로우로 DB 설계를 하면 됨.

우리는 시행사도 여러 개이고, 가구 업체도 여러 개이다.
각각의 관리자가 있고, 총괄 관리자가 있도록.
